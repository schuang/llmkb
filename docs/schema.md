# Wiki Schema

This document describes the on-disk schema and file formats used by the `llmkb` engine to manage a personal knowledge base.

It acts as both:
- a reference for editing or debugging the repository
- a contract for the tools reading and writing `artifacts/` and `wiki/`

## Design Principles

- **Immutability:** `raw/` remains the immutable source of truth for original PDFs.
- **Determinism:** `artifacts/` stores deterministic machine-readable state generated by the engine.
- **Browseability:** `wiki/` stores markdown pages intended for Obsidian-style browsing.
- **Provenance:** Page-level provenance is preferred whenever evidence is shown.
- **Stable References:** `[[source/<doc_id>]]`, `[[source/<doc_id>#p123]]`, and `[[concepts/<concept_id>]]` are the primary internal link forms.

## Naming Conventions

### `doc_id`
A stable document identifier derived from the source filename, refined by overrides when needed.
Example: `canuto-spectral-methods`

### `concept_id`
A URL-safe slug for a concept page.
Example: `spectral-method`

## Link Schema

### Source Links
- **Source page:** `[[source/<doc_id>]]`
- **Page anchor:** `[[source/<doc_id>#p<page_number>]]` (1-based index)
  - Example: `[[source/canuto-spectral-methods#p115]]`

### Concept Links
- **Concept page:** `[[concepts/<concept_id>]]`
  - Example: `[[concepts/spectral-method]]`

## Artifact Schemas

### `artifacts/catalog/sources.json`
Purpose: Canonical machine-readable catalog of the raw corpus.

**Key Fields:**
- `doc_id`: Stable identifier (usually matching the filename stem).
- `filename`: Name of the physical file on disk.
- `path`: Relative path from the KB root.
- `doi`: Digital Object Identifier (for papers).
- `isbn`: International Standard Book Number (for books).
- `metadata_source`: Source of the metadata (e.g., `crossref`, `openlibrary`, `datacite`, `heuristic`).
- `sha256`: SHA-256 content hash for duplicate detection.
- `source_class`: Classification (e.g., `book`, `paper`, `manual`, `duplicate`).

```json
{
  "schema_version": 1,
  "source_classes": ["book", "paper", "notes"],
  "stats": {
    "document_count": 216,
    "duplicate_group_count": 4,
    "class_counts": {
      "book": 180
    }
  },
  "documents": [
    {
      "doc_id": "canuto-spectral-methods",
      "filename": "2006-canuto-spectral-methods.pdf",
      "path": "raw/2006-canuto-spectral-methods.pdf",
      "year": 2006,
      "sha256": "...",
      "size_bytes": 123456,
      "page_count": 481,
      "title": "Spectral Methods",
      "author": "Claudio Canuto, M. Y. Hussaini, Alfio Quarteroni, Thomas A. Zang",
      "doi": null,
      "isbn": "978-3-540-30704-4",
      "metadata_source": "openlibrary",
      "producer": "PDF producer string",
      "source_class": "book",
      "canonical_doc_id": "canuto-spectral-methods",
      "relationship": null,
      "wiki_link": "[[source/canuto-spectral-methods]]",
      "aliases": ["2006-canuto-spectral-methods.pdf"],
      "notes": []
    }
  ]
}
```

**Allowed relationship kinds:** `exact_duplicate_of`, `near_duplicate_of`, `excerpt_of`, `companion_to`, `derived_from`, `supersedes`

### `artifacts/extract/<doc_id>/metadata.json`
Purpose: Extraction status and provenance for a single source.

```json
{
  "doc_id": "canuto-spectral-methods",
  "status": "extracted",
  "source_sha256": "...",
  "source_path": "raw/2006-canuto-spectral-methods.pdf",
  "page_count": 481,
  "pdf_page_count": 481,
  "text_bytes": 1234567,
  "quality_flags": [],
  "updated_at": "2026-04-05T04:16:20+00:00"
}
```

### `artifacts/extract/<doc_id>/pages.json`
Purpose: Page-level extracted text used for source pages, concepts, and retrieval.

**Key Fields:**
- `page_number`: Integer for paginated PDFs, or a string slug (e.g., `"introduction"`) for reflowable formats like ePub/Markdown.
- `preview`: Short snippet of text from the beginning of the page/section.
- `text`: Full extracted text.

```json
{
  "doc_id": "canuto-spectral-methods",
  "pages": [
    {
      "page_number": 115,
      "char_count": 1024,
      "preview": "5.1 The Steady Navier-Stokes Equations",
      "text": "Full extracted page text..."
    }
  ]
}
```

### `artifacts/compile/source_resolution.json`
Purpose: Exact and near-duplicate resolution mapping used during generation.

```json
{
  "schema_version": 1,
  "updated_at": "2026-04-05T04:16:23+00:00",
  "near_duplicate_count": 3,
  "cluster_count": 1,
  "clusters": [],
  "documents": [
    {
      "doc_id": "alternate-doc",
      "canonical_doc_id": "canonical-doc",
      "status": "near_duplicate",
      "relationship": {
        "kind": "near_duplicate_of",
        "target_doc_id": "canonical-doc"
      },
      "similarity": 0.91,
      "reasons": ["very_high_text_similarity"]
    }
  ]
}
```

### `artifacts/compile/search_index.json`
Purpose: Retrieval index over source pages. Designed to be sufficient for ranking without re-reading wiki markdown.

```json
{
  "schema_version": 2,
  "updated_at": "2026-04-05T04:16:24+00:00",
  "documents": [
    {
      "doc_id": "canuto-spectral-methods",

      "source_class": "book",
      "canonical_doc_id": "canuto-spectral-methods",
      "wiki_link": "[[source/canuto-spectral-methods]]",
      "summary": "This book is developed from lecture notes of graduate courses...",
      "keywords": ["spectral method", "navier-stokes equation", "galerkin method"],
      "relationships": [],
      "page_count": 481,
      "page_previews": [
        {
          "page_number": 115,
          "preview": "5.1 The Steady Navier-Stokes Equations"
        }
      ],
      "chapters": [
        {
          "chapter_number": 5,
          "title": "Navier-Stokes Equations",
          "page_start": 213,
          "page_end": 284,
          "summary": "Chapter-level summary...",
          "keywords": ["navier-stokes equation", "incompressible flows", "spectral method"],
          "wiki_link": "[[source/canuto-spectral-methods#p213]]",
          "section_previews": ["The Unsteady Navier-Stokes Equations", "Steady Navier-Stokes Equations"]
        }

      "search_text": "internal retrieval field",
      "is_redundant": false,
      "dedupe_kind": null
    }
  ]
}
```

### `artifacts/compile/concept_index.json`
Purpose: Retrieval index over concept pages. Concept pages must aggregate at least two distinct supporting sources.

```json
{
  "schema_version": 1,
  "updated_at": "2026-04-05T04:16:26+00:00",
  "concepts": [
    {
      "concept_id": "spectral-method",
      "title": "Spectral Method",
      "wiki_link": "[[concepts/spectral-method]]",
      "summary": "This concept aggregates sources related to spectral method.",
      "keywords": ["spectral", "method"],
      "source_count": 10,
      "supporting_sources": [
        {
          "doc_id": "canuto-spectral-methods",
          "title": "Spectral Methods",
          "wiki_link": "[[source/canuto-spectral-methods]]",
          "source_class": "book",
          "score": 37.0,
          "pages": [
            {
              "page_number": 115,
              "score": 9.0,
              "preview": "5.1 The Steady Navier-Stokes Equations",
              "link": "[[source/canuto-spectral-methods#p115]]"
            }
          ]
        }
      ],
      "related_concepts": ["[[concepts/finite-element-method]]"],
      "search_text": "Title, keywords, summary, and source titles..."
    }
  ]
}
```

## Wiki Page Schemas

### `wiki/source/<doc_id>.md`
Purpose: One page per canonical source document.

**Frontmatter:**
```yaml
---
doc_id: "canuto-spectral-methods"
title: "Spectral Methods"
author: "Claudio Canuto, M. Y. Hussaini, Alfio Quarteroni, Thomas A. Zang"
year: 2006
source_class: "book"
canonical_doc_id: "canuto-spectral-methods"
keywords:
  - "spectral method"
  - "navier-stokes equation"
aliases:
  - "2006-canuto-spectral-methods.pdf"
related: []
generated_at: "2026-04-05T04:16:24+00:00"
---
```

**Body sections:**
- `# <Title>`
- `## Summary`
- `## Keywords`
- `## Chapters` (when structure is available)
- `## Metadata`
- `## Relationships` (if relationships exist)
- `## Page Anchors` (e.g. `### p115`)

### `wiki/concepts/<concept_id>.md`
Purpose: Topic-level aggregation across multiple sources.

**Frontmatter:**
```yaml
---
concept_id: "spectral-method"
title: "Spectral Method"
keywords:
  - "spectral"
  - "method"
supporting_sources:
  - "[[source/canuto-spectral-methods]]"
source_count: 10
related:
  - "[[concepts/finite-element-method]]"
generated_at: "2026-04-05T04:16:26+00:00"
---
```

**Body sections:**
- `# <Title>`
- `## Summary`
- `## Key Sources`
- `## Supporting Pages`
- `## Related Concepts`

### `wiki/syntheses/*.md`
Purpose: Higher-level notes, comparisons, and research outputs. This area is extensible for future LLM-authored research notes.

**Frontmatter:**
```yaml
---
kind: "comparison"
query: "spectral method"
generated_at: "2026-04-05T04:16:13+00:00"
involved_sources:
  - "[[source/canuto-spectral-methods]]"
status: "evidence-pack"
---
```

## Invariants & Change Policy
- Every source has one canonical `doc_id`.
- Markdown pages are views over the artifact state; the artifact state is rebuilding-safe from `raw/` plus `config/`.
- If a JSON artifact breaks compatibility, the producer tool should bump its `schema_version`.
